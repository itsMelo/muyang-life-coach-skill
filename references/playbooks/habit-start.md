# Playbook — 起步 / 拖延 / 完美主义循环

**触发**：迟迟不开始（自媒体、 side project、健身、发第一条内容）、「永远在准备」、怕认真做证明自己不行。

## 核心机制（常一起出现）

1. **失败 = 身份审判**（不是「这次没做好」，是「我不行」）
2. **高代价记忆放大低赌注任务**（过去买房/婚姻/职业失败 → 把小实验当人生终审）
3. **INTJ/完美主义**：准备阶段安全，发布/承诺 = 暴露

## 流程

### 1. 分离赌注

问：「如果这次 **彻底失败**，你实际会失去什么？用时间和钱估算。」

对照 profile 收入/存款 → 说出容错月数。明确：「这是 C 级实验，不是 A 级人生判决。」

### 2. 身份句柄

若用户说「每篇都要代表我」→ 重构：「第一篇是 **采样**，不是 **判决书**。转行程序员时你也是边做边学——那次怎么开始的？」

### 3. 最小发布单元

共同定义 **不可再拆** 的一步（例：300 字 + 1 图、15 分钟录屏、发给一个朋友而非公开）。

### 4. 时间窗口

用 profile `精力与约束` 里的 **最清醒时段** 排执行意图，勿安排在精力谷底。

### 5. Conclude 默认

**行为激活** + 情绪前后 1–10 分：

```yaml
type: behavior_activation
hypothesis: "发出/完成 [最小单元] 后焦虑会下降，且不会证明我不行"
action: "[具体时间] [具体动作]"
due: YYYY-MM-DD
metrics:
  anxiety_before: null
  anxiety_after: null
```

若用户追近型（关系模式）：预判负面反馈时的 **一条** 解离句预备好。

### 6. 跟进

下次 session 或 weekly review **必须先读** 该条 experiment 结果；skipped 则进 patterns「准备型拖延」。
